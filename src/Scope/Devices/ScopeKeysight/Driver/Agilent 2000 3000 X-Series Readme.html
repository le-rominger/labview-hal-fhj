3. Known Issues

  • Configure Arbitrary Waveform Data.vi
    • While using commands in this VI(WGEN:ARB:DATA and WGEN:ARB:DATA:DAC) as Programmer's Guide described, error "-222, Data out of range" is reported. These two commands are swapped in the VI to avoid error report, but the functionality are not fully tested. Configure Serial Bus (LIN - Trigger - Symbolic) and Configure Serial Bus (CAN - Trigger - Symbolic) commands implemented, but the functionality are not fully tested.
    Default Values are based on the latest added model or model series (As of 12/2017 VIs that are supported by the 1000 Series will have 1000 series default values).
    To report issues or provide feedback about this instrument driver, please send an email to instrument.drivers@ni.com.

    REV 1.3.1, 09/16/2015
    Modified by: Jun Zhang, Shanghai.
    Modified VI:
  • Public/Data/Low Level/Fetch Waveform.vi.
    • Previously the VI had a subtle contradiction: the label said 'Data encoding (U16)', which suggested that the encoding method should be U16; but the command sent was ':WAV:FORM BYTE', which meant that the encoding method actually used was U8.
    Data encoding method was changed from BYTE/U8 to WORD/U16 to fix the inconsistency. Also type cast is modified accordingly. This would increase precision and fix the data low resolution issue.
    If previous behavior (using BYTE/U8) is preferred, please follow the comments in Fetch Waveform.vi to change it.
